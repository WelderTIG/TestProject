Test SPA (Angular)

Цель:

Создание приложения, для отладки игрового баланса PvP.

1 ЭТАП

Создание набора персонажей, который будет доступен в приложении.

    Персонажи должны иметь две основные характеристики:
  • Сила
    • Первая производная - урон (далее - DMG)
    • Вторая производная - количество здоровья (далее - HP)
  • Ловкость
    • Первая производная - скорость атаки (далее - AS)
    • Вторая производная - броня (далее - ARM)

2 ЭТАП

Создание структуры проекта.

Project details:

Описание функционала приложения.

Описание сущностей Angular, с помощью которых реализован функционал.

Home page:

Создание двух списков персонажей с отображением основных характеристик. Выбор соперников. Отображение производных характеристик.

Fight:

Запуск поединка между соперниками. Поединок осуществляется броском кубика с 6 (шестью) гранями (от 1 до 6 целыми числами).

    Работают производные характеристики:
  • 1) Успешная атака : кубик A -(минус) кубик B > 0
  • 2) Результат успешной атаки HP = HP - (DMG - ARM)
  • 3) Бой начинается персонажем с большей ловкостью. Бой должен проходить в реальном времени. Т.е. каждый персонаж атакует с учетом своей скорости атаки (AS)

Бой заканчивается когда у одного из персонажей здоровье падает до HP меньше 0.

Функция проведения N-поединков. Запись статистики.

Stats:

На основании собранных данных построение графиков и предоставление иной информации.

Базовый график: ось X - сражения. Ось Y : (+) - количество HP оставшееся у стороны А. (-) - количество HP оставшееся у стороны B.

  • Осуществление передачи данных из Fighters.ts в Home page. Затем из Home page в Fight. Затем из Fight в Stats.
